# 🧩 Guia de Contribuição — Guias Rápidos

Obrigado por considerar contribuir com este repositório!  
Este documento descreve as diretrizes para colaboração, organização e envio de melhorias.

---

## 📌 1. Como posso contribuir?

Existem várias formas de contribuir:

- Correções de erros de digitação ou formatação  
- Melhorias nos textos dos guias  
- Sugestões de novos tópicos  
- Inclusão de exemplos práticos  
- Revisão técnica  
- Organização estrutural do repositório  
- Criação de novos guias

---

## 🐛 2. Reportando problemas (Issues)

Se encontrar um problema, abra uma **Issue** contendo:

1. **Título claro e objetivo**  
2. **Descrição detalhada**  
3. Se aplicável, indique:
   - Arquivo afetado  
   - Trecho específico  
   - Sugestão de correção  

Evite abrir issues duplicadas — verifique antes se já existe algo relacionado.

---

## 🔧 3. Enviando melhorias (Pull Requests)

Para enviar um PR:

### 1. Faça um **fork** do repositório

### 2. Crie uma branch descritiva

```bash
git checkout -b docs/melhoria-guia-01
```

### ⚡ 3. Realize as alterações

### 4. Garanta que o texto esteja

- Claro  
- Coeso  
- Sem erros de digitação  
- Formatado corretamente  

### 5. Envie o PR com

- Descrição objetiva  
- Motivação da mudança  
- Arquivos alterados listados  

PRs pequenos e focados são mais fáceis de revisar.

---

## 🧱 4. Padrão de Commits

Use mensagens de commit claras e consistentes.  
Sugestão de convenção:

- `docs:` para alterações em documentação  
- `feat:` para novos conteúdos  
- `fix:` para correções  
- `refactor:` para reorganização sem mudança de conteúdo  
- `chore:` para ajustes menores

Exemplos:

```bash
docs(guia01): corrige explicação sobre git add
feat(guia02): adiciona seção sobre git commit --amend
```

---

## 📚 5. Estrutura dos Guias

Para manter consistência, cada guia deve seguir:

- Título claro  
- Introdução breve  
- Conteúdo organizado em seções  
- Exemplos reais  
- Comandos formatados em blocos de código  
- Conclusão ou resumo  
- Linguagem simples e objetiva  

---

## 🧪 6. Qualidade e Revisão

Antes de enviar contribuições:

- Revise ortografia  
- Verifique links  
- Teste comandos (quando aplicável)  
- Mantenha o estilo visual consistente  
- Evite jargões desnecessários  

---

## 🤝 7. Código de Conduta

Ao contribuir, você concorda em seguir o **CODE_OF_CONDUCT.md** deste repositório, garantindo um ambiente respeitoso e colaborativo.

---

## 📬 8. Dúvidas

Se tiver dúvidas sobre como contribuir, abra uma Issue com o prefixo:

`[question]`

ou entre em contato diretamente pelo GitHub.

---

Obrigado por ajudar a melhorar os **Guias Rápidos**!
